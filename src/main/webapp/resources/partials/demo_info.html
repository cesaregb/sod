Demo Info

Application Engineer Coding Exercise

  • Create a simple web app, hosted at a URL we can visit
  • Users can enter a URL of a page to fetch
  • The web app fetches the HTML of the page and displays the source to the user
  • A summary of the document is displayed, listing which tags are present in the HTML and how many of each tag
  • Clicking on the name of each tag in the summary will highlight the tags in the source code view

Proposed solution

Client Tier:
Angular JS + Boostrap
Business Tier:
Java + Jersey
Data Tier:
No information beins saved, Usage of Redis can be incorporated, this way avoid making the http request and handling the cache-like Non-SQL database.
UI Info:
Default service (No checkbox selected). This default service have the logic for:
* Making the http request com.example.service.URLReader
* Parsing the html content, this contains some logic to detect the thml tags, generate tha parse version of the text com.example.util.ParserHelper
Please Note that this parser is not a full DOM validator, so soveral script in-code tags could be detected as "normal" tags.

Jsoup Checkbox, if selected, Applicatoin will consume the jsoup web service, this uses the Java Library JSOUP used to parse and handle xml/html.
